# Условные выражения и сопоставление с образцами
---
## 1. Условные выражения

**Синтаксис:** `if cond then trueBranch else falseBranch`

Условное выражение вычисляет условие `cond`. Если оно истинно (`true`), выполняется `trueBranch`, иначе — `falseBranch`. Обе ветви должны возвращать значения **одного типа**.

### Особенности условных выражений:
* Выражение `if‑then‑else` всегда должно содержать ветку `else`.
* Тип результата определяется типом выражений в `then` и `else`.
* Условия могут быть произвольными выражениями типа `Bool`.

### Примеры использования:

```bash
if 5 > 3 then "больше" else "меньше или равно"
// Результат: "больше"
```
```bash
let isPositive = \n ->
  if n > 0 then true else false
in isPositive (-5)
// Результат: false
```
```bash
if true then 42 else 0
// Результат: 42
```

## 2. Сопоставление с образцами (match)

Конструкция `match` позволяет деструктурировать списки и выполнять разные действия для пустого и непустого списка.

## Базовый синтаксис

```bash
match список with
| nil -> результат_для_пустого_списка
| cons(head, tail) -> результат_для_непустого_списка
```

## Правила сопоставления

- Порядок ветвей фиксирован: сначала `nil`, затем `cons(head, tail)`.
- Выполняется ветвь, соответствующая форме списка.
- В текущей версии поддерживаются только ветви `nil` и `cons(head, tail)`.
- Обе ветви списка должны быть обработаны явно.

Примеры сопоставления с образцами:

**1. Сопоставление со списками:**

```bash
match cons(1, cons(2, nil)) with
| nil -> "пустой список"
| cons(head, tail) -> "не пустой список"
// Результат: "не пустой список"
```
```bash
match nil with
| nil -> "пустой список"
| cons(head, tail) -> "не пустой список"
// Результат: "пустой список"
```

**2. Сопоставление с простыми значениями не поддерживается.**
```bash
if 3 == 3 then "среда" else "другой день"
// Результат: "среда"
```

**3. Сопоставление с кортежами не поддерживается.**

```bash
let pair = (true, 42) in
if fst pair then snd pair else 0
// Результат: 42
```

**4. Сопоставление с вложенными списками возможно через вложенный `match`.**

```bash
match [1; 2] with
| nil -> "список пуст"
| cons(head, tail) -> head
// Результат: 1
```

**5. Охранные условия (guards) не поддерживаются.**

```bash
let x = 7 in
if x > 0 && x < 10 then "положительное однозначное" else "другое"
// Результат: "положительное однозначное"
```
---
## Сравнение `if` и `match`

| Критерий | `if` | `match` |
|----------|------|---------|
| **Назначение** | Проверка булевых условий | Деструктуризация списков |
| **Гибкость** | Ограничена логическими условиями | В текущей версии ограничена списками |
| **Читаемость** | Хороша для простых условий | Лучше для разбора списков |
| **Исчерпываемость** | Всегда определена (есть `else`) | Требует ветки `nil` и ветки `cons` |
