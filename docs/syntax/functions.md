# Функциональное программирование: ключевые концепции


## Лямбда‑выражения

**Лямбда‑выражение** — это анонимная функция, определяемая в месте использования.

**Синтаксис:** `\параметр -> тело_функции`

**Пример:**
```bash
\x -> x * 2 // Функция, удваивающая входное значение
```


**Примеры использования:**

1. Прямое применение:
```bash
(\x -> x * 2) 5 // Результат: 10
```


2. Передача в качестве аргумента:
```bash
map (\x -> x + 1) [1; 2; 3] // Результат: [2; 3; 4]
```


3. Сохранение в переменной:
```bash
let double = \x -> x * 2 in double 7 // Результат: 14
```

## Применение функций

Функции применяются в **префиксной форме** с пробелом между именем функции и аргументами.

**Синтаксис:** `f x y z`

**Особенности:**
* **Левоассоциативность применения:** `f x y z` эквивалентно `((f x) y) z`.
* Это позволяет использовать **каррирование** — частичное применение функций.

**Примеры:**
```bash
// Простая функция сложения
let add = \x -> \y -> x + y in

// Полное применение
let result = add 3 5 in // Результат: 8

// Частичное применение — создаём функцию «прибавить 3»
let addThree = add 3 in
addThree 5 // Результат: 8
```

## Рекурсия

Рекурсия — основной механизм организации повторяющихся вычислений в функциональном программировании (вместо циклов).

**Для определения рекурсивных функций используется `let rec`.**

**Пример: вычисление факториала**

```bash
let rec factorial = \n ->
    if n == 0 then 1
    else n * (factorial (n - 1))
in factorial 5 // Результат: 120
```


**Разбор примера:**
1. `let rec factorial` — объявляем рекурсивную функцию `factorial`.
2. `\n -> ...` — функция принимает один параметр `n`.
3. Базовый случай: `if n == 0 then 1` — факториал 0 равен 1.
4. Рекурсивный случай: `else n * (factorial (n - 1))` — умножаем `n` на факториал `(n - 1)`.

**Другие примеры рекурсии:**

* Вычисление чисел Фибоначчи:
```bash
let rec fib = \n ->
    if n <= 1 then n
    else fib (n - 1) + fib (n - 2)
in fib 6
```

* Суммирование списка:
```bash
let rec sumList = \lst ->
    match lst with
    | nil -> 0
    | cons(x, xs) -> x + (sumList xs)
in sumList [1; 2; 3]
```

## Замыкания (Closures)

**Замыкание** — функция, которая «запоминает» переменные из области видимости, в которой она была создана.

**Ключевая особенность:** функция сохраняет доступ к переменным окружения даже после выхода из этой области видимости.

**Пример 1: простая фабрика функций**

```bash
let makeMultiplier = \factor ->
\x -> x * factor
in
let double = makeMultiplier 2 in
let triple = makeMultiplier 3 in

double 5 // Результат: 10
```

**Объяснение:**
* `makeMultiplier` возвращает лямбда‑выражение `\x -> x * factor`.
* Внутри этой лямбды переменная `factor` — свободная (не определена в теле функции).
* При создании замыкания значение `factor` «захватывается» и сохраняется.

**Пример 2: независимые замыкания**
```bash
let makeAdder = \initial ->
    \x -> initial + x
in
let add1 = makeAdder 1 in
let add100 = makeAdder 100 in
add1 5 // Результат: 6
```

**Объяснение:**
* Переменная `initial` доступна только внутри функции, которую вернул `makeAdder`.
* Возвращаемая функция имеет доступ к `initial` через замыкание.
* Каждый вызов `makeAdder` создаёт своё собственное замыкание.

**Важные свойства замыканий:**
* Позволяют создавать **функции с сохранённым окружением**.
* Обеспечивают **инкапсуляцию** (скрытие данных).
* Используются для реализации **частичного применения** и **каррирования**.
* Являются основой для многих паттернов функционального программирования.
